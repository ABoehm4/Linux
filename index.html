How to hide Apache Version and OS Identity from Errors

When Apache is installed on your server, it will display the Apache version that is installed on your operating system. You can test this by typing in http://"Your server address"/test, with what your server IP address is in quotes.

When you type in the previous html command, the following page will show up (with your IP information).

As you can see, the Apache version and your server address are displayed for all to see.

In order to hide that information, we are going to make some changes to the Apache main config file.
Open the main config file with your preferred text editor: /etc/httpd/conf/httpd.conf (RHEL/CenOS/Fedora) and /etc/apache2/apache2.conf (Debian/Ubuntu)

nano /etc/httpd/conf/httpd.conf
nano /etc/apache2/apache2.conf

Search for "ServerSignature", it should be set to On by default. We are now going to turn that setting Off.
You also need to add the line ServerTokens Prod if it's not already there. This will tell Apache to return only Apache as its product and not return the operating system or server address.

ServerSignature Off
ServerTokens Prod

Once you changed the first line to Off, and added the second line, save and exit the main config file.
Now type in server http restart (RHEL/CentOS/Fedora) service apache2 restart (Debian/Ubuntu). You can now retest your webpage and the information should not be displayed.

Keep updating Apache regularly

Apache developers are constantly working on fixing security issues and other general fixes for Apache. They consistently release new versions of Apache with new security options. It is recommended that we keep Apache updated with its newest version.
To check your Apache version, type in httpd -v command. It should come back with your Server version and your Server built date.

httpd -v

Server version: Apache/2.2.15 (Unix)
Server built: Aug 13 2013 17:29:28

You can update your Apache server by entering the command yum update http. You should also ensure you are keeping your Kernel and OS update with the most recent stable builds.

Use Allow and Deny to Restrict access to Directories

You can restrict access to directories with Allow and Deny options in the main configuration file /etc/httpd/conf/httpd.conf. For this example, we'll secure our root directory.
Open your main configuration file and search for Directory /

</Directory />
Options None
Order deny,allow
Deny from all
</Directory>

  • Options "None" - This will not allow other users to enable any optional features.
  • Order deny, allow - This is the order in which the directives of Deny and Allow are processed. For this example, it is Deny first, allow second.
  • Deny from all - This denies all requests from everybody to the root directory.
